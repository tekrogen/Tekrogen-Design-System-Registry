# 01 · Rebrand the Registry to the Tekrogen Brand

A methodical, **token-first** plan to migrate the registry-starter defaults to the
Tekrogen Brand Design System — shipped in **small, independently-validated PRs**,
each gated by tests and the 3-expert UI/UX review.

> Pairs with `00.install-shadcn-registry.md` (how the registry is built/served)
> and the repo's governance in `CLAUDE.md` (Rule #4: the validation gate).

## Brand principles to enforce (non-negotiable)

- **Token-only.** Every color/size references a `--tk-*` token (mapped to a shadcn
  var). No hardcoded hex or px in components.
- **Dark is default**, **Paper** is the light override (`[data-tk-theme="paper"]`).
- **Sans-only**: Poppins (display/headings/body), Manrope (secondary), JetBrains
  Mono (eyebrows/meta/code). Self-hosted woff2 — no font CDN.
- **Hairline borders, not drop shadows** (`--tk-border*`); shadows are rare.
- **AA always**: lowest text color is `--tk-fg-3`; `fg-4`/`fg-5` are decorative
  only. 12px legibility floor (`--tk-fs-eyebrow`/`--tk-fs-meta` = `0.75rem`).
- **rem type scale** (fluid `clamp`), signature tracking, 18° tilt for the mark.
- **Incremental**: one surface/concern per PR; never a big-bang rebrand.

## Testing & validation model (applies to every phase)

Three layers — the first two are automated (devflow model), the third is the
human/agent gate from `CLAUDE.md` Rule #4.

1. **Vitest (unit/contract)** — `tests/**/*.test.ts`, node env, `@`→`src`.
   - Token contract: the `theme` item's `cssVars` contain the expected Tekrogen
     values; no hardcoded px; nothing below the 12px floor; lowest text token is
     `fg-3`. Snapshot the built `public/r/theme.json`.
2. **Playwright e2e (`tests/e2e`, chromium, `webServer: pnpm dev`)** — assert
   *rendered* truth, not source:
   - computed `font-family` is Poppins; `--background`/`--foreground` resolve to
     the brand ink/paper values; focus ring uses `--tk-focus`.
   - **zero horizontal overflow at 390px** (`scrollWidth === clientWidth`).
   - **axe** accessibility scan (AA) on each surface; dark **and** paper themes.
   - capture screenshots at **1440px + 390px** for the review.
3. **MCP Playwright (interactive) + 3-expert review** — drive the running app,
   eyeball the brand, and run the 3-expert panel (Senior Product Designer ·
   Design Systems Architect · Front-End Eng Lead). Record an evidence-backed
   verdict in `admin/review/VALIDATION-<phase>.md`. **Serve, never `file://`.**

**Per-phase "done" = all three pass** (green Vitest + Playwright, clean 3-expert
note) before the PR merges.

## Phase 1 · Foundations — tokens & fonts

This is the highest-leverage phase: get the tokens right and most components
inherit the brand for free.

- [ ] Self-host the brand fonts: copy Poppins/Manrope/JetBrains Mono woff2 into
      `public/fonts/` (or wire `next/font/local`); remove any Google Fonts links.
- [ ] Port the Tekrogen palette + type/spacing/radii/motion tokens into
      `src/app/globals.css`, keyed to shadcn variable names (mapping below).
      Dark = default (ink); add the **Paper** override.
- [ ] Update the `theme` registry item in `registry.json` (replace the starter's
      "Nature Theme") so installs carry Tekrogen `cssVars` (light = paper,
      dark = ink) + font vars.
- [ ] `pnpm registry:build`; confirm `/tokens` renders the Tekrogen palette/type.
- **Test:** Vitest token-contract spec + `theme.json` snapshot; Playwright checks
      computed background/foreground/font on `/` and `/tokens`, both themes.
- **Validate:** 3-expert pass on `/tokens` at 1440 + 390 → `VALIDATION-phase1.md`.
- **Ship:** `feat: adopt Tekrogen brand tokens and fonts`

## Workflow per task (the loop)

```
branch  feat/<issue#>-<slug>            (from up-to-date main; see CLAUDE.md)
build   make the token/component change
test    pnpm test   &&   pnpm e2e        (Vitest + Playwright green)
gate    static token audit → render 1440/390 → 3-expert note in admin/review/
PR      conventional title; "Closes #<issue>"; pr.yml CI must pass
ship    squash-merge → release-please cuts the version
```

## Acceptance bar (every phase, from CLAUDE.md Rule #4)

1. **Static token audit** — every `font-size` is a type-scale token (no px); none
   below the 12px floor; lowest text color is `--tk-fg-3` (never fg-4/5).
2. **Render check** — 1440px + 390px; **zero horizontal overflow at 390px**;
   served (not `file://`); dark **and** paper.
3. **3-expert pass** — evidence-backed verdict recorded in `admin/review/`.
